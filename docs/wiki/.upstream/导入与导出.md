# 导入与导出

## 支持导入的格式

根据项目公开说明，当前支持的导入来源包括：

- Bitwarden JSON
- Bitwarden CSV
- Bitwarden 密码库 + 附件 ZIP
- NodeWarden JSON
- 网页导入器中显示的其他浏览器或密码管理器格式

其中最值得注意的是“Bitwarden 密码库 + 附件 ZIP”。这是很多迁移用户真正需要的能力，因为现实里最麻烦的往往不是密码条目本身，而是那些历史附件。

## 支持导出的格式

当前支持的导出方式包括：

- Bitwarden JSON （可加密）
-  Bitwarden 密码库 + 附件 ZIP（可加密）
- NodeWarden JSON （可加密）
- 备份中心里的实例级完整手动导出

这意味着你不仅能把数据导出来，而且能根据后续去向选择更合适的格式。

## 建议的迁移顺序

如果你是从官方 Bitwarden 或其他兼容服务迁移过来，建议按下面顺序操作：

1. 先导出原有密码库
2. 在 NodeWarden 新实例中导入
3. 检查条目数量、文件夹、附件和 TOTP 字段
4. 用至少两个客户端做一次同步验收
5. 再决定是否切换为主用实例

别在没有核对的情况下，直接把旧实例停掉。

## 导入后重点核对什么

建议至少检查：

- 条目总数是否一致
- 文件夹是否完整
- 收藏状态是否保留
- 附件是否都能打开
- TOTP 是否正常
- Send 是否按预期存在

如果你有历史上很久以前导入过来的特殊条目，也建议抽样检查几条，尤其是包含较多自定义字段的记录。

## 导出不等于备份

这是一个很容易被忽略的点。

导出更多是“拿出一份当前数据副本”，而备份中心解决的是“持续、定时、面向灾难恢复”的问题。两者最好同时使用，而不是互相替代。

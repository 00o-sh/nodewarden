# 功能总览

这页不只是列清单，更想把 NodeWarden 当前“真正能用到什么程度”讲清楚。

## 1. 网页密码库

NodeWarden 提供自己的 Web Vault 界面，不只是一个 API 外壳。你可以直接在网页端完成常见管理操作，包括查看、编辑和整理密码库内容。

对于很多自托管项目来说，网页端往往只是“有就行”；但 NodeWarden 的网页端是正式组成部分，前端也独立维护在 `webapp/` 目录里。

## 2. 兼容官方客户端同步

项目明确支持 Bitwarden 官方客户端的同步流程，尤其是 `/api/sync` 兼容能力。这意味着你不需要重新迁移使用习惯，也不需要接受一个功能残缺的第三方客户端。

这部分兼容性，是 NodeWarden 能真正落地使用的基础。

## 3. 附件上传与下载

附件支持是 NodeWarden 非常实用的一项能力。

你可以根据部署方式选择：

- R2 作为默认附件存储
- KV 作为附件回退方案

如果你只是保存少量文本型凭据，KV 也能用；但如果你会保存带附件的条目，R2 更值得优先考虑。

## 4. Send

NodeWarden 支持 Bitwarden 的 Send 能力，包括：

- 文本 Send
- 文件 Send

这类功能在日常里很容易被低估，但一旦你已经在用 Bitwarden 生态，Send 往往会成为临时共享密码、便签和小文件的高频入口。

## 5. 导入与导出

这是项目非常有分量的一部分。

当前已支持的导入来源包括：

- Bitwarden JSON
- Bitwarden CSV
- Bitwarden 密码库加附件 ZIP
- NodeWarden JSON
- 网页导入器中列出的多种浏览器或密码管理器格式

支持的导出方式包括：

- Bitwarden JSON
- Bitwarden 加密 JSON
- 带附件 ZIP 导出
- NodeWarden JSON 系列
- 备份中心中的实例级完整手动导出

这意味着 NodeWarden 不只是“能存”，而是把“迁入”和“迁出”都认真考虑过了。

## 6. 云端备份中心

这是 NodeWarden 非常值得单独拿出来说的一项功能。

项目支持将实例备份到：

- WebDAV
- E3

而且不只是手动备份，还支持定时备份。对于密码管理服务来说，这项能力的重要性并不亚于同步本身。

## 7. TOTP 与 Steam TOTP

项目支持常规 TOTP，同时也支持 `steam://` 相关场景。

这意味着它并不是只追求“最基本兼容”，而是在一些典型使用细节上也做了延展，实际体验会更接近“一个可以长期用的密码库服务端”。

## 8. 多用户与邀请码注册

NodeWarden 支持多用户，不是单用户实验项目。管理员可以通过邀请码控制新用户注册，这样既保留了多用户能力，又避免把公开注册入口一直暴露出去。

对于家庭共享、小型协作或小圈子自用，这种模式已经足够实用。

## 9. 用户级登录 2FA

当前项目对登录二次验证的支持是“部分支持”，重点是用户级 TOTP。

如果你对登录安全有更高要求，建议在实际部署时把这一项作为重点验证内容，而不是默认认为它和官方企业方案完全等价。

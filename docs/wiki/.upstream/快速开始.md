# 快速开始

这页给的是最短路径，目标只有一个：让你尽快把 NodeWarden 跑起来。

如果你是第一次部署，建议先用默认的 R2 模式；如果你的 Cloudflare 账号暂时没有开通 R2，或者不想绑卡，也可以直接使用 KV 模式。

## R2 和 KV 该怎么选

   | 储存 | 是否需绑卡 | 单个附件/Send文件上限 | 免费额度 |
   |---|---|---|---|
   | R2 | 需要 | 100 MB（软限制可更改） | 10 GB |
   | KV | 不需要 | 25 MiB（Cloudflare限制） | 1 GB |


## 部署前准备

开始之前，你需要准备好下面这些东西：

- 一个 GitHub 账号
- 一个 Cloudflare 账号
- 一双能动的手（单手也可以）

## 最省事的方式：网页一键部署

这是最适合大多数用户的路径。

1. Fork `NodeWarden` 仓库到自己的 GitHub 账号
2. 进入 Cloudflare Workers 创建页面
3. 选择 `Continue with GitHub`
4. 选择你刚刚 Fork 的仓库
5. 保持默认配置继续部署
6. 如果你打算用 KV 模式，把部署命令改成 `npm run deploy:kv`
7. 等部署完成后，打开生成的 Workers 域名
8. 根据页面提示设置`JWT_SECRET` ，不建议临时乱填。这个值直接关系到令牌签发安全，正式环境至少使用 32 个字符以上的随机字符串。

## 命令行部署

如果你更习惯本地开发或手动发布，可以走 CLI：

```powershell
git clone https://github.com/shuaiplus/NodeWarden.git
cd NodeWarden
npm install
npx wrangler login

# 默认使用 R2
npm run deploy

# 可选：KV 模式
npm run deploy:kv
```

本地开发，或cloudflare崩溃时，可在本地运行demo来恢复密码

```powershell
npm run dev
npm run dev:kv
```

## 升级与同步上游

如果你使用 Fork 仓库部署，后续升级最自然的方式就是同步上游代码。

- 手动同步 `Sync fork -> Update branch`
- 启用仓库 Actions 中的 `Sync upstream` 工作流

如果你不改源码，自动同步会很省心；如果你做了自己的定制，建议保持手动同步，这样更容易在升级前确认变更。

# 常见问题

## 部署与绑定

### 重新部署后为什么 `JWT_SECRET` 会“失效”

`JWT_SECRET`应该被设置为密钥，而不是文本！

### KV 部署时报绑定 ID 不存在，或者示例里的 KV ID 对不上

这个报错的根源通常不是代码坏了，而是把示例里的 `kv_namespaces.id` 当成了可以直接复用的真实资源 ID。

KV Namespace ID 是你自己账号下创建出来的资源 ID，不会和别人仓库示例完全相同。issue #61 就是这个问题：看到配置里有固定 ID，手动创建了 KV 以后发现 ID 不一样，于是误以为部署方式有问题。

处理方式很简单：

- 自己创建 KV Namespace
- 把配置绑定到你自己账号下的那个实际 ID
- 不要照抄示例里仓库作者账号的资源 ID

 
而不是“附件记录全保留，点开再说”。
